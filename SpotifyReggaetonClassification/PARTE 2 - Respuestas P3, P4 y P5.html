XGBost usa cmo estndar un corte de 0.5 para assignar cada registra a la clase positiva (en este caso reggaeton). Si bien este corte podra ser modificado, dado que no se tienen costos asociados a los errores de missclasification, no podramos determinar un ptimo en ese sentido. Es por esto que se decidi dejar el estndar de 0.5 cmo punto de corte.
